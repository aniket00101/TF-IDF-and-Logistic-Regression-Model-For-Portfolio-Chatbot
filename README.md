# 🤖 Aniket's Portfolio Chatbot — Backend

Intent-based chatbot using **TF-IDF + Logistic Regression**, served via **FastAPI**.

---

## 📁 Project Structure

```
chatbot-backend/
│
├── dataset/
│   └── training_data.csv        ← 300 labeled training sentences (text, intent)
│
├── model/
│   └── train_chatbot.ipynb      ← Jupyter notebook: preprocess → train → evaluate → save
│
├── trainedmodel/
│   └── intent_model.pkl         ← Saved Pipeline (TF-IDF + LR) — generated by notebook
│
├── main.py                      ← FastAPI server (loads .pkl, serves /chat endpoint)
├── portfolio_data.py            ← All portfolio content (projects, skills, education, etc.)
├── tech_info.py                 ← Detailed descriptions for specific technology questions
├── Chatbot.jsx                  ← React frontend component (calls FastAPI via fetch)
├── requirements.txt             ← All Python dependencies
└── README.md
```

## 🚀 Getting Started

### 1. Create and activate virtual environment
```bash
python -m venv venv

# Windows
venv\Scripts\activate

# Mac / Linux
source venv/bin/activate
```

### 2. Install dependencies
```bash
pip install -r requirements.txt
```

### 3. Register venv as a Jupyter kernel (run once)
```bash
pip install ipykernel
python -m ipykernel install --user --name=venv --display-name "Python (venv)"
```

### 4. Open the Jupyter notebook and train the model
```bash
jupyter notebook model/train_chatbot.ipynb
```
- Select kernel → **Python (venv)**
- Run all cells → saves `trainedmodel/intent_model.pkl`

### 5. Start the FastAPI server
```bash
uvicorn main:app --reload --port 8000
```

### 6. Test the API
```bash
curl -X POST http://localhost:8000/chat \
  -H "Content-Type: application/json" \
  -d '{"message": "what skills do you have"}'
```

Expected response:
```json
{
  "reply": "Here's Aniket's full tech stack:\n\n🎨 Frontend : ...",
  "intent": "skills",
  "confidence": 0.91
}
```

## ☁️ Deploy to Render (Free)

1. Push this folder to a **new GitHub repo**
   > ⚠️ Make sure `trainedmodel/intent_model.pkl` is committed — Render won't run the notebook.

2. Go to [render.com](https://render.com) → **New Web Service** → connect repo

3. Fill in these settings:

   | Field | Value |
   |---|---|
   | Runtime | Python 3 |
   | Region | Singapore (closest to India) |
   | Build Command | `pip install -r requirements.txt` |
   | Start Command | `uvicorn main:app --host 0.0.0.0 --port $PORT` |

4. Choose **Free** plan → Click **Create Web Service**

5. Copy your Render URL (e.g. `https://aniket-chatbot.onrender.com`)

6. In your React project root, create a `.env` file:
   ```
   VITE_CHATBOT_API_URL=https://aniket-chatbot.onrender.com
   ```

> 💤 **Free tier note:** Render spins down after 15 min of inactivity. The first request after that takes ~30 seconds to wake up — this is normal.

## 🧠 How It Works

```
User message
     ↓
 preprocess()         lowercase, remove punctuation, lemmatize, remove stopwords
     ↓
 TF-IDF Vectorizer    convert text → numeric feature vector (unigrams + bigrams)
     ↓
 Logistic Regression  predict intent + confidence score across 13 intents
     ↓
 confidence < 0.32?   → fallback response
     ↓
 build_response()     return portfolio-specific reply from portfolio_data.py
     ↓
 JSON response        { reply, intent, confidence }
```

## 📂 Key Files Explained

**`portfolio_data.py`** — Edit this file to update any portfolio content (new projects, new experience, contact info, certificates). No need to retrain the model for content changes.

**`tech_info.py`** — Edit this file to add or update technology descriptions. Covers 24 technologies including React, Node.js, Python, MongoDB, WebRTC, GraphQL, and more.

**`dataset/training_data.csv`** — The labeled training data with 300 rows across 13 intents. Edit this to improve or add intent recognition.

**`model/train_chatbot.ipynb`** — Full training pipeline with data visualization, model evaluation, confusion matrix heatmap, smoke test, and model saving.

## ➕ Adding New Intents

1. Add new rows to `dataset/training_data.csv`:
   ```csv
   tell me a joke,jokes
   say something funny,jokes
   ```

2. Re-run `model/train_chatbot.ipynb` (all cells) → new `.pkl` is saved

3. Add response logic in `main.py` inside `build_response()`:
   ```python
   if intent == "jokes":
       return "Why do programmers prefer dark mode? Because light attracts bugs! 😄"
   ```

4. Restart the server:
   ```bash
   uvicorn main:app --reload --port 8000
   ```
